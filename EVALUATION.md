# Qwopus 성능 평가

> Qwen3.5-27B-Claude-4.6-Opus-Reasoning-Distilled (Q5_K_M) 기준
> 32K 컨텍스트, 듀얼 GPU (16GB x2)

## 작업별 성능

| 작업 | 성능 | 설명 |
|------|------|------|
| 단일 파일 읽고 설명 | ⭐⭐⭐⭐ | 파일 하나 읽고 요약/설명하는 건 잘함 |
| 단일 스크립트 작성 | ⭐⭐⭐⭐ | 100줄 이내 스크립트 생성 능력 양호 |
| 단일 파일 리팩토링 | ⭐⭐⭐ | 파일이 컨텍스트에 들어가면 괜찮음 |
| 버그 디버깅 (단일 파일) | ⭐⭐⭐ | 에러 메시지 + 소스 보고 원인 추론 가능 |
| DB 쿼리 결과 정리 | ⭐⭐⭐⭐ | 구조화된 데이터 → 표/요약은 LLM 강점 |
| 셸 명령어 조합 | ⭐⭐⭐⭐ | grep, awk, sed 등 파이프라인 잘 만듦 |
| 설정 파일 생성 | ⭐⭐⭐⭐ | yaml, json, toml 등 잘 생성 |
| 멀티파일 리팩토링 | ⭐⭐ | 32K 컨텍스트로 3~4개 파일이 한계 |
| 대규모 프로젝트 구조 파악 | ⭐⭐ | 인덱서로 보완하지만 깊은 이해는 어려움 |
| 복잡한 아키텍처 설계 | ⭐⭐ | 단순 설계는 가능, 복잡한 의존성은 놓침 |
| 긴 대화 유지 | ⭐⭐ | 컨텍스트 트림 시 이전 맥락 유실 |

## Hallucination 평가

| 상황 | 위험도 | 대응 |
|------|--------|------|
| 없는 함수/API 이름 생성 | 🟡 중간 | Grep으로 검증하라는 시스템 프롬프트로 완화 |
| 요청 안 한 파일 생성 | 🟢 낮음 | 시스템 프롬프트로 제어됨 |
| 같은 도구 반복 호출 | 🟢 낮음 | 반복 감지로 자동 중단 |
| 파일 내용 추측 | 🟡 중간 | "파일 읽기 전 참조 금지" 규칙으로 완화 |
| 주제 이탈 | 🟢 낮음 | 인덱서 + 포커스 가이드로 완화 |
| 긴 대화 후 맥락 혼동 | 🔴 높음 | 컨텍스트 한계 — /compact 또는 /clear로 대응 |
